# 페르소나 시뮬레이션: 인천 유나이티드 응원가 웹페이지

- **문서 버전**: v1.0
- **작성일**: 2026-07-11
- **연관 문서**: PRD.md (v1.0), 페르소나 프로필.md (v1.0)
- **목적**: 각 페르소나가 PRD에 정의된 기능(F-01 목록 / F-02 상세 / F-03 모바일)을 실제로 사용하는 흐름을 따라가 보며, 잘 맞는 부분과 부족한 부분을 미리 발견합니다.

## 시뮬레이션 읽는 법
- **상황**: 이 사람이 왜 웹을 켰는가
- **행동 흐름**: 화면에서 실제로 하는 동작을 순서대로
- **판정**: PRD 기능이 이 사람을 만족시켰는지 (✅ 충족 / ⚠️ 아쉬움)

---

## 시뮬레이션 1. 열성팬 김도훈 (P1)

### 상황
경기 전, 신입 팬 후배에게 '선수 응원가'를 보여주려고 스마트폰으로 웹을 켰습니다.

### 행동 흐름
1. 웹 접속 → **목록 화면(F-01)** 이 뜬다.
2. 응원가 카드들이 세로로 나열되어 있다.
3. 원하는 곡을 찾기 위해 목록을 스크롤한다.
4. 해당 곡을 탭 → **상세 화면(F-02)** 진입.
5. 유튜브 영상과 가사를 후배에게 보여준다. → 목표 달성.

### 판정
- ✅ (F-01) 목록에서 곡을 훑어보는 흐름은 문제없이 작동합니다.
- ✅ (F-02) 영상+가사로 후배에게 보여주는 목적은 충족됩니다.
- ⚠️ **발견된 아쉬움**: 곡이 20~30개로 늘어나면, 원하는 곡을 찾기 위해 한참 스크롤해야 합니다. 김도훈의 '몇 초 안에 찾고 싶다'는 목표는 MVP만으로는 절반만 충족됩니다.
- ➡️ **시사점**: 확장 기능 **F-11(검색)** 의 우선순위를 높게 잡아야 함을 확인했습니다.

## 시뮬레이션 2. 신규팬 이서연 (P2) — ⭐핵심 타겟

### 상황
집에서 다음 직관을 준비하며, 응원가를 미리 연습하려고 웹을 켰습니다.

### 행동 흐름
1. 웹 접속 → **목록 화면(F-01)** 이 뜬다.
2. 어떤 곡부터 봐야 할지 몰라, 일단 맨 위 곡을 탭한다.
3. **상세 화면(F-02)** 에서 유튜브 영상을 재생해 멜로디를 듣는다.
4. 영상 아래 **가사(F-02)** 를 보며 작은 소리로 따라 부른다.
5. '뒤로 가기'로 목록에 돌아가 다음 곡을 반복 학습한다.

### 판정
- ✅ (F-02) **영상+가사 동시 제공**이라는 이 웹의 핵심 가치가 이서연에게 정확히 꽂힙니다. 가장 이상적인 사용 사례입니다.
- ✅ (F-03) 스마트폰 세로 화면에서 영상과 가사가 편하게 보입니다.
- ⚠️ **발견된 아쉬움**: '어떤 곡부터 봐야 할지 모른다'는 막막함이 남습니다. 목록이 그냥 나열만 되어 있으면 입문 순서를 알 수 없습니다.
- ➡️ **시사점**: 확장 기능 **F-12(카테고리/태그)** 로 '대표곡'을 표시하면 입문자 경험이 크게 좋아집니다. MVP에서는 최소한 **목록 정렬 순서(대표곡을 위쪽에)** 만 신경 써도 도움이 됩니다.

## 시뮬레이션 3. 관심 입문자 박준영 (P3)

### 상황
동네 팀이 궁금해, 별생각 없이 링크를 눌러 웹에 처음 들어왔습니다.

### 행동 흐름
1. 웹 접속 → **목록 화면(F-01)** 이 뜬다.
2. (로그인 요구가 없어서 안심하고) 화면을 훑어본다.
3. 눈에 띄는 곡 하나를 탭한다.
4. **상세 화면(F-02)** 에서 영상을 재생해 응원 분위기를 구경한다.
5. 영상 1~2개만 보고 만족한 뒤 창을 닫는다.

### 판정
- ✅ (Non-Goals) **로그인이 없어** 이탈하지 않고 진입에 성공합니다. PRD의 '회원가입 없음' 결정이 이 사람을 살렸습니다.
- ✅ (F-03) 화면이 단순해 기술 친숙도가 낮은 박준영도 헤매지 않습니다.
- ⚠️ **발견된 아쉬움**: 박준영은 '가사'보다 '영상 분위기'가 목적입니다. 상세 화면에서 영상이 **가사보다 위/먼저** 보이는 배치가 이 사람에게 유리합니다.
- ➡️ **시사점**: 상세 화면 레이아웃을 **'영상 먼저, 가사 아래'** 로 두는 것이 세 페르소나 모두에게 무난합니다. (이서연도 멜로디부터 듣고 싶어 함)

## 시뮬레이션 종합 결론

| 항목 | 결과 |
|------|------|
| MVP 핵심 기능(영상+가사)이 핵심 타겟(P2)에게 잘 맞는가 | ✅ 매우 적합 |
| 로그인 없는 단순 구조 결정이 옳았는가 | ✅ P3가 증명 |
| MVP만으로 부족한 부분은 | ⚠️ 검색(F-11), 대표곡 구분(F-12) |

### PRD에 반영하면 좋을 제안
1. **상세 화면 배치 확정**: '유튜브 영상 → 가사' 순서로 (세 페르소나 공통 이득)
2. **F-11(검색)** 을 확장 기능 중 **1순위**로 명시
3. MVP 단계에서도 **목록 정렬 기준**(예: 대표곡·인기곡 위쪽)을 간단히라도 정해두기

> 위 3가지는 코드를 크게 바꾸지 않고도 반영 가능한 수준입니다. 지금 PRD에 메모해두면 개발 단계의 혼선을 줄일 수 있습니다.
