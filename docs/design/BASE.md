## 1. 비주얼 콘셉트 (Theme & Atmosphere)

이 서비스는 **'검정 바탕 위에서 파랑이 빛나는'** 다크 테마입니다. 인천 유나이티드의 정체성인 **파랑-검정(파랑검정)**을 그대로 화면 언어로 삼았습니다.

- **검정은 무대, 파랑은 조명입니다.** 배경은 파랑기가 살짝 도는 짙은 검정으로 깔고, 영상 썸네일·가사·응원가 카드가 콘텐츠로서 화면을 채웁니다.
- **파랑(인천 블루)은 장식이 아니라 '기능'입니다.** 재생 버튼, 선택된 탭, 대표곡 배지, 주요 버튼 등 **사용자의 행동·상태를 나타낼 때만** 씁니다. 그냥 예뻐 보이려고 파랑을 뿌리지 않습니다.
- **단, '파랑검정' 정체성을 위한 세로 스트라이프 악센트는 예외입니다.** 카드·영상·가사 등 콘텐츠 블록 왼쪽에 붙는 얇은 세로 바는 장식 목적으로 파랑/검정을 번갈아 씁니다. (4장 "스트라이프 악센트" 참고)
- **경기장(야외)에서도 잘 보이게.** 글자 대비를 충분히 확보하고, 터치 영역을 크게 둡니다. (PRD 접근성 요구사항)

### 핵심 특징 요약
- 파랑기 도는 near-black 다크 테마 (`#0E1116`~`#1E232C`)
- 인천 블루(`#2E7DFF`)를 **유일한 기능적 강조색**으로 사용
- 콘텐츠 블록마다 **파랑/검정 세로 스트라이프 악센트**를 번갈아 배치해 '파랑검정' 정체성을 시각적으로 드러냄
- 한글 최적화 폰트 **Pretendard**
- 둥근 pill 버튼 + 원형 아이콘 버튼 (터치 친화)
- 가사는 **읽기 편한 넉넉한 행간**으로 (Spotify와 다른 점 — 아래 4·5장 참고)

## 2. 색상 팔레트 (Color Palette)

### 브랜드 / 강조
- **인천 블루** (`#2E7DFF`): 주 강조색. 재생 버튼, 선택된 탭, 주요 CTA, 활성 상태
- **인천 블루 딥** (`#0B4DA2`): 로고·큰 면적 채움 등 정체성 표현용 진한 파랑
- **블루 호버** (`#1F6BEA`): 버튼을 누르거나 올렸을 때

> ⚠️ 위 파랑 값은 화면용으로 잡은 **근사치**입니다. 구단의 공식 브랜드 컬러 hex를 확보하면 이 값을 교체해 주세요.

### 배경 (어두울수록 깊은 층)
- **베이스** (`#0E1116`): 가장 깊은 페이지 배경
- **표면** (`#161A21`): 카드·컨테이너 배경
- **인터랙티브** (`#1E232C`): 버튼·탭 등 누를 수 있는 면
- **카드 강조** (`#232935`): 살짝 떠 있는 카드

### 텍스트
- **화이트** (`#FFFFFF`): 기본 텍스트, 제목
- **실버** (`#A8B0BD`): 보조 텍스트, 비활성 항목
- **뮤트** (`#6B7480`): 흐린 안내문, 미세 정보

### 테두리 / 구분선
- **보더** (`#2E3542`): 기본 테두리
- **보더 라이트** (`#414A5A`): 강조 테두리, 구분선

### 시맨틱 (상태 알림)
- **네거티브 레드** (`#F3727F`): 오류·경고 상태
- **워닝 오렌지** (`#FFA42B`): 주의 상태
- (정보 알림은 브랜드 파랑을 절제해서 재사용)

## 3. 타이포그래피 (Typography)

### 글꼴
- **폰트**: `Pretendard`
- **폴백**: `-apple-system, BlinkMacSystemFont, "Apple SD Gothic Neo", "Malgun Gothic", "맑은 고딕", sans-serif`

> **왜 Pretendard인가**: 이 서비스는 제목·가사 등 **한글 텍스트가 중심**입니다. Pretendard는 한글·영문·숫자가 고르게 예쁘고, 무료로 웹에서 바로 쓸 수 있어 한국어 UI의 사실상 표준입니다.

### 위계 (Hierarchy)

| 역할 | 크기 | 굵기 | 행간 | 비고 |
|------|------|------|------|------|
| 화면 제목 | 24px | 700 | 1.3 | 예) "인천 유나이티드 응원가" |
| 응원가 제목 | 18px | 700 | 1.35 | 상세 화면 곡 제목 |
| 카드 제목 | 16px | 600 | 1.4 | 목록의 곡 제목 |
| **가사 본문** | 17px | 400 | **1.75** | 넉넉한 행간 (따라 부르기 편하게) |
| 버튼 | 15px | 600 | 1.0 | 탭·주요 버튼 |
| 배지/태그 | 12px | 700 | 1.4 | '대표곡' 등 |
| 미세 정보 | 12px | 400 | 1.5 | 안내문, 출처 |

### 원칙
- **굵기 대비로 위계를 만듭니다.** 대부분 700(강조)과 400(본문)을 씁니다. 크기를 잔뜩 늘리기보다 굵기 차이로 정리합니다.
- **가사는 예외적으로 넉넉하게.** Spotify는 목록을 빠르게 훑는 앱이라 촘촘하지만, 우리 서비스의 **가사는 '읽고 따라 부르는' 콘텐츠**입니다. 그래서 가사만큼은 크기(17px)와 행간(1.75)을 넉넉히 줍니다. *(원본과의 의도적 차이)*
- **버튼 라벨은 한글이라 대문자·자간 규칙을 쓰지 않습니다.** (Spotify의 영문 대문자 버튼 스타일은 한글에 어울리지 않으므로 제거) *(의도적 차이)*

## 4. 컴포넌트 (Components)

### 스트라이프 악센트 (전 화면 공통) — '파랑검정' 정체성
- 응원가 카드, 영상 영역, 가사 영역 등 **콘텐츠 블록 왼쪽**에 붙는 두께 8px, 완전히 둥근(pill) 세로 바
- 블록 본문과는 12px(`gap-3`) 간격을 두고 별도 요소로 배치 (본문 배경·모서리는 건드리지 않음)
- 색상은 **인천 블루(`#2E7DFF`)와 보더 라이트(`#414A5A`, '검정' 대표색)를 번갈아** 사용
  - 응원가 카드 목록: 카드 순서(0부터)가 짝수면 파랑, 홀수면 보더 라이트
  - 상세 화면: 영상 영역은 파랑 고정, 가사 영역은 보더 라이트 고정
- 검정 스트라이프에 순검정(`#000000`)이 아닌 보더 라이트를 쓰는 이유: 배경(`#0E1116`~`#1E232C`)과 구분되어 실제로 눈에 보여야 하기 때문

### 응원가 카드 (목록 화면 F-01)
- 배경: `#161A21`, 모서리: 12px, 테두리 없음
- 곡 제목: 16px / 600 / 화이트
- **대표곡 배지**: 인천 블루(`#2E7DFF`) 배경, 화이트 텍스트, 12px, 둥근 pill
- 누를 때(hover/press): 배경이 살짝 밝아짐(`#1E232C`)
- 왼쪽에 스트라이프 악센트 동반 (위 "스트라이프 악센트" 참고)

### 영상 선택 탭 (F-04) — 세그먼트 pill
- 컨테이너: `#1E232C` 배경, 9999px(완전 둥근 pill)
- **선택된 탭**: 인천 블루(`#2E7DFF`) 배경 + 화이트 텍스트
- **선택 안 된 탭**: 투명 배경 + 실버(`#A8B0BD`) 텍스트
- 예: `[ 공식 영상 | 현장 영상 ]` — 한 번에 하나만 강조

### 영상 영역 (F-02)
- 16:9 비율 유지, 화면 너비에 꽉 맞춤(반응형)
- 모서리: 12px, 배경(로딩 중): `#161A21`
- 왼쪽에 파랑 스트라이프 악센트 고정 (위 "스트라이프 악센트" 참고)

### 가사 영역 (F-02)
- 배경: `#0E1116`(베이스와 동일해 텍스트에 집중)
- 텍스트: 17px / 400 / 화이트, 행간 1.75, 가운데 정렬
- 줄바꿈은 원본 그대로 표시
- 왼쪽에 보더 라이트(검정) 스트라이프 악센트 고정 (위 "스트라이프 악센트" 참고)

### 뒤로 가기 버튼 (F-02)
- 원형 아이콘 버튼: `#1E232C` 배경, 화이트 아이콘, 50% radius, 안쪽 여백 12px

### 주요 버튼 (Primary)
- 인천 블루(`#2E7DFF`) 배경, 화이트 텍스트, 9999px pill, 여백 10px 20px
- 호버/누름: `#1F6BEA`

### 빈 상태 / 안내 문구
- 뮤트(`#6B7480`) 텍스트로 중앙 정렬
- 예: "등록된 응원가가 없습니다", "영상 준비 중입니다"

## 5. 레이아웃 (Layout)

### 기준: 모바일 우선 (Mobile-First)
- **한 손 세로 화면**을 기본으로 설계합니다. 태블릿·PC에서는 콘텐츠 폭을 가운데로 제한(최대 약 480~640px)해 너무 넓게 퍼지지 않게 합니다.
- Spotify의 사이드바·하단 재생바 같은 **앱형 요소는 사용하지 않습니다.** 우리 서비스는 목록 → 상세의 단순한 흐름이기 때문입니다. *(의도적 차이)*

### 간격 시스템
- 기본 단위: 8px
- 자주 쓰는 값: 4, 8, 12, 16, 20, 24, 32px

### 여백 철학
- **콘텐츠 위주, 적당한 숨 쉴 틈.** 목록은 카드 사이 여백을 두어 손가락으로 정확히 누르게 하고, 가사는 넉넉한 여백으로 읽기 편하게 합니다. (Spotify처럼 극단적으로 촘촘하게 하지 않습니다.)

### 모서리 둥글기 스케일 (Radius)
- 4px: 배지·작은 요소
- 8px: 입력창·작은 버튼
- 12px: 카드·영상 영역 (표준)
- 9999px: pill 버튼·세그먼트 탭
- 50%: 원형 아이콘 버튼

## 6. 깊이 & 그림자 (Depth & Elevation)

| 단계 | 처리 | 사용처 |
|------|------|--------|
| 베이스 | `#0E1116` 배경 | 페이지 바탕 |
| 표면 | `#161A21` | 카드·컨테이너 |
| 떠 있음 | `rgba(0,0,0,0.4) 0px 6px 16px` | 눌린 카드·떠 있는 요소 |
| 모달 | `rgba(0,0,0,0.55) 0px 8px 24px` | 오버레이(추후 필요 시) |

- **어두운 배경에서는 그림자를 충분히 진하게.** 옅은 그림자는 검정 위에서 보이지 않으므로, 0.4~0.55 정도의 진한 그림자를 씁니다.

## 7. Do / Don't

### Do
- 배경은 파랑기 도는 near-black(`#0E1116`~`#1E232C`)으로, 명도 차이로 층을 만든다
- 인천 블루(`#2E7DFF`)는 **재생·선택·주요 버튼·대표곡 배지** 등 기능에만 쓴다
- 콘텐츠 블록(카드·영상·가사) 왼쪽에는 파랑/검정 세로 스트라이프 악센트를 번갈아 배치해 '파랑검정' 정체성을 드러낸다
- 버튼은 pill(9999px), 아이콘 버튼은 원형(50%)으로 둥글게
- 가사는 크게(17px) + 넉넉한 행간(1.75)으로 읽기 편하게
- 어두운 배경 위 그림자는 진하게(0.4~0.55)

### Don't
- 파랑을 배경 전체나 장식용으로 남발하지 않는다 (기능 전용 + 스트라이프 악센트만 예외)
- 밝은 흰 배경을 주요 화면 바탕으로 쓰지 않는다 (다크 몰입감이 정체성)
- 파랑·검정 외에 새 브랜드 색을 추가하지 않는다 (시맨틱 색은 예외)
- 버튼을 각지게 만들지 않는다 (둥근 기하가 정체성)
- 한글 버튼에 영문식 대문자·넓은 자간을 쓰지 않는다
- 가사에 촘촘한 행간을 쓰지 않는다 (따라 부르기 불편)
- 스트라이프 악센트를 블록 본문 배경이나 텍스트 위에 겹쳐 그리지 않는다 (가독성 저하 방지, 항상 별도 세로 바로만 표현)

## 8. 반응형 (Responsive)

| 이름 | 폭 | 주요 변화 |
|------|-----|-----------|
| 모바일 | < 600px | 기본. 1열, 콘텐츠 꽉 채움 |
| 태블릿 | 600–900px | 콘텐츠 폭 중앙 제한 |
| PC | > 900px | 콘텐츠 폭 최대 약 640px로 중앙 배치 |

- 영상은 모든 크기에서 16:9 유지, 폭에 맞춰 조절
- 어떤 화면에서도 **가로 스크롤이 생기지 않게** 한다

## 9. AI 에이전트용 프롬프트 가이드 (Agent Prompt Guide)

> AI(Stitch, Claude Code 등)에게 화면을 시킬 때 붙여 쓰기 좋은 요약입니다.

### 빠른 색상 참조
- 배경: `#0E1116` / 표면: `#161A21` / 인터랙티브: `#1E232C`
- 텍스트: `#FFFFFF` / 보조: `#A8B0BD`
- 강조(인천 블루): `#2E7DFF` / 호버: `#1F6BEA`
- 테두리: `#2E3542` / 보더 라이트(스트라이프 '검정'): `#414A5A` / 오류: `#F3727F`
- 폰트: `Pretendard`

### 예시 프롬프트
- "다크 카드 만들기: `#161A21` 배경, 12px 모서리. 곡 제목 16px Pretendard 600 화이트. 대표곡 배지는 `#2E7DFF` 배경 pill."
- "영상 선택 탭(세그먼트): `#1E232C` 배경 pill(9999px). 선택된 탭은 `#2E7DFF` 배경 화이트, 비선택은 투명 배경 `#A8B0BD` 텍스트."
- "가사 영역: `#0E1116` 배경, 17px Pretendard 400 화이트, 행간 1.75, 가운데 정렬, 줄바꿈 유지."
- "뒤로 가기 버튼: `#1E232C` 배경, 화이트 아이콘, 원형(50%), 여백 12px."
- "스트라이프 악센트: 블록 왼쪽에 8px 두께의 완전히 둥근 세로 바, 12px 간격. 카드는 순서에 따라 `#2E7DFF`/`#414A5A` 번갈아, 영상 영역은 `#2E7DFF` 고정, 가사 영역은 `#414A5A` 고정."
