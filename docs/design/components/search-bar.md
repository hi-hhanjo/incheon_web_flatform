# 컴포넌트: SearchBar (검색창)

- **문서 버전**: v1.1 · **작성일**: 2026-07-17 (v1.1 갱신: 2026-07-17)
- **연관 문서**: BASE.md (5장 Radius 8px '입력창', 4장 '빈 상태/안내 문구'), FUNCTION.md (F-11, v1.5), lyrics-snippet.md, feedback.md
- **한 줄 요약**: 응원가를 **제목·가사**로 걸러내는 입력창. 입력할 때마다 목록이 즉시 좁혀집니다.

---

## 1. 개요
- 목록 화면(F-01)에서 곡이 34개가 되며 "스크롤로 눈으로 찾기"가 힘들어져 만든 부품입니다(FUNCTION.md F-11).
- **검색 버튼이 없습니다.** 데이터가 로컬 JSON이라 입력 즉시 걸러집니다 — 기다릴 네트워크가 없어 버튼은 탭 한 번만 늘립니다.
- 이 부품은 **검색어를 위로 올리는 것까지만** 합니다. "무엇을 남길지"(걸러내기)와 "어떤 순서로 놓을지"(정렬)는 SongList 책임입니다. *(v1.1 — 이전엔 "정렬은 SortToggle 담당"이라 적혀 있었으나 그 부품은 제거됐습니다. FUNCTION.md v1.5)*
- **검색 결과의 근거를 보여주는 일은 LyricsSnippet이 합니다** *(v1.1)*. 가사에 걸린 곡은 카드 아래에 걸린 가사 한 줄이 붙습니다 — 이 부품은 거기에 관여하지 않습니다.

## 2. 사용처
- F-01 목록 화면, **화면 제목과 카드 목록 사이** (곡이 1개 이상일 때만 표시)

## 3. 입력 (Props)
| 이름 | 설명 | 필수 |
|------|------|------|
| query | 현재 검색어 | O |
| onChange | 글자가 바뀔 때마다 호출 (지우기 버튼은 `""`로 호출) | O |

## 4. 동작
1. 글자를 입력하면 **입력할 때마다** `onChange`가 호출된다. (엔터·버튼 불필요)
2. 검색어가 비어 있지 않으면 오른쪽에 **지우기(×) 버튼**이 나타난다. 누르면 `onChange("")`.
3. 곡이 0개(빈 상태)면 이 컴포넌트 자체를 렌더링하지 않는다.
4. **걸러내는 규칙은 화면(SongList) 책임**이며, 이 부품은 검색어만 위로 올린다.
   - 대상: `title` 또는 `lyrics`
   - 대소문자 무시, 앞뒤 공백 무시 (공백만 입력 = 검색 안 한 것)

## 5. 스타일 규칙 (BASE.md 기준)
- 컨테이너: `#1E232C` 배경(인터랙티브 — 누를 수 있는 면), 모서리 **8px**(BASE.md 5장 '입력창')
- 입력 글자: 15px / 400 / 화이트
- 안내 문구(placeholder): 뮤트(`#6B7480`) — BASE.md 4장 '빈 상태/안내 문구'와 동일 색
- 높이: 최소 44px (F-03 터치 영역)
- 지우기 버튼: 실버(`#A8B0BD`) × 아이콘, 최소 44px 터치 영역
- **인천 블루를 테두리·배경에 쓰지 않습니다.** 검색창은 '상태'가 아니라 '입력 그릇'이라 BASE.md 1장의 "파랑은 기능(선택·재생·CTA)에만" 원칙 밖입니다. 포커스 표시는 브라우저 기본 외곽선을 살립니다.

## 6. 접근성
- `<input type="search">`를 쓰고 `aria-label="응원가 검색"`을 붙인다. (화면에 라벨 글자가 없으므로)
- 지우기 버튼에 `aria-label="검색어 지우기"`.

## 7. 사용 예시 (의사코드)
```
<SearchBar query={검색어} onChange={(q) => 검색어변경(q)} />
```

## 8. AI 지시용 스니펫
