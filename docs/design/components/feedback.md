# 컴포넌트: Feedback (로딩·빈 상태·에러 안내)

- **문서 버전**: v1.1 · **작성일**: 2026-07-11
- **연관 문서**: BASE.md, FUNCTION.md (F-01, F-02, F-04)
- **한 줄 요약**: 화면이 비거나, 불러오는 중이거나, 문제가 생겼을 때 사용자에게 보여주는 안내 표시를 한곳에 모은 컴포넌트.

---

## 1. 개요
- 정상 콘텐츠가 없을 때 **화면이 하얗게/까맣게 비는 것을 막는** 안내 컴포넌트입니다.
- 세 가지 종류를 하나로 관리합니다: **로딩 / 빈 상태 / 에러**.

## 2. 사용처 (기능명세서의 예외 상황들)
| 종류 | 언제 | 문구 예시 |
|------|------|-----------|
| loading | 데이터 불러오는 중 (F-01) | "불러오는 중..." |
| empty | 곡이 하나도 없음 (F-01) | "등록된 응원가가 없습니다" |
| empty | 영상 없음 (F-04) | "영상 준비 중입니다" |
| empty | 가사 없음 (F-02) | "가사 준비 중입니다" |
| error | 없는 곡 접근 (F-02) | "해당 응원가를 찾을 수 없습니다" |

## 3. 입력 (Props)
| 이름 | 설명 | 필수 |
|------|------|------|
| type | loading / empty / error | O |
| message | 보여줄 문구 | O |
| action | (선택) 버튼 — 예: "목록으로 가기" | X |

## 4. 스타일 규칙 (BASE.md 기준)
- 문구: `#6B7480` (뮤트) / 14~15px / Pretendard
- **중앙 정렬**, 위아래 여백 넉넉히
- loading: 간단한 텍스트 또는 원형 스피너(`#2E7DFF`)
- error에 버튼이 붙는 경우 → `button.md`의 Primary 버튼 사용

## 5. 동작
- 스스로 동작은 없고, 상위 화면이 상황에 맞는 type과 message를 넣어 보여줍니다.
- error 타입에서 `action`이 있으면 그 버튼을 눌러 목록 등으로 이동할 수 있습니다.

## 6. 사용 예시 (의사코드)
```
// 목록이 비었을 때
<Feedback type="empty" message="등록된 응원가가 없습니다" />

// 없는 곡에 접근했을 때
<Feedback
  type="error"
  message="해당 응원가를 찾을 수 없습니다"
  action={<Button label="목록으로 가기" onClick={목록으로} />}
/>
```

## 7. AI 지시용 스니펫
