# 컴포넌트: LyricsSnippet (가사 스니펫)

- **문서 버전**: v1.0 · **작성일**: 2026-07-17
- **연관 문서**: BASE.md (2장 색, 3장 위계, 5장 Radius), FUNCTION.md (F-11, v1.5), song-card.md, search-bar.md, lyrics.md
- **한 줄 요약**: 가사가 검색어에 걸린 곡의 **카드 바로 아래**에 붙어, 걸린 가사 한 줄을 검색어 강조와 함께 보여주는 작은 카드.

---

## 1. 개요
- **왜 있나**: 가사로 검색하면 제목만 적힌 카드가 나와, 팬이 "내가 기억하는 소절이 이 곡 맞나"를 확인하려면 상세 화면까지 들어갔다 나와야 했습니다(FUNCTION.md F-11 v1.5). 근거가 된 가사 한 줄을 목록에서 바로 보여줘 그 왕복을 없앱니다.
- **검색 중에만 존재합니다.** 검색어가 없으면 렌더링되지 않습니다 — 목록의 기본 모습은 F-01 그대로입니다.
- **Lyrics(가사 표시 영역)와 다른 부품입니다.** Lyrics는 상세 화면에서 **따라 부르라고** 가사 전문을 크게(17px/1.75, 가운데 정렬) 보여줍니다. 이 부품은 목록에서 **곡을 식별하라고** 한 줄만 작게(15px, 왼쪽 정렬) 보여줍니다 — 목적이 달라 규격을 공유하지 않습니다.
- **무엇을 강조할지 스스로 정하지 않습니다.** 어느 줄을 어떻게 자를지는 `lib/search.ts`가 정하고, 이 부품은 받은 조각을 그리기만 합니다.

## 2. 사용처
- F-01 목록 화면, **SongCard 바로 아래** (검색어가 있고, 그 곡의 **가사**에 걸렸을 때만)
- 제목에만 걸린 곡에는 붙지 않습니다. 카드 제목이 이미 근거를 보여주기 때문입니다.

## 3. 입력 (Props)
| 이름 | 설명 | 필수 |
|------|------|------|
| segments | 가사 한 줄을 검색어 기준으로 쪼갠 조각 목록 (`{ text, matched }[]`) | O |

- `matched: true`인 조각이 강조 대상입니다. 한 줄에 검색어가 여러 번 나오면 조각도 여러 개입니다.

## 4. 동작
1. `segments`를 순서대로 이어 붙여 **한 줄로** 그린다. (원래 가사의 줄바꿈은 이미 잘려 나간 상태다)
2. `matched` 조각은 `<mark>`으로 감싸 강조하고, 나머지는 보조 텍스트 색으로 둔다.
3. 줄이 화면 폭을 넘으면 **줄바꿈해서 감싼다.** 가로 스크롤을 만들지 않는다(F-03).

## 5. 스타일 규칙 (BASE.md 기준)
- 컨테이너: `#161A21` 배경(표면 — 카드보다 한 단계 뒤로 물러난 면), 모서리 **8px**, 위쪽 여백 4px로 카드에 붙인다
- 본문: 15px / 400 / 실버(`#A8B0BD`) — 카드 제목(16px/600/화이트)보다 약하게 두어 **카드가 주인공**임을 유지한다
- 강조(`<mark>`): 인천 블루(`#2E7DFF`) 배경 + 화이트 글자, 모서리 4px
  - **파랑을 쓰는 근거**: BASE.md 1장은 파랑을 "사용자의 행동·상태를 나타낼 때만" 쓰라고 정합니다. 여기서 강조는 **"당신이 친 검색어가 여기 걸렸다"는 상태 표시**이므로 이 원칙 안입니다. 장식이 아닙니다.
  - 브라우저 기본 `<mark>` 스타일(노란 배경·검은 글자)은 다크 테마와 팔레트 밖이므로 **반드시 덮어씁니다.**
- **터치 대상이 아닙니다.** 링크·버튼이 아니므로 44px 규칙(F-03)이 적용되지 않습니다. 곡으로 이동하려면 위의 카드를 누릅니다.

## 6. 접근성
- `<mark>`은 시맨틱 강조 태그라 그대로 씁니다. (색만으로 강조하지 않고 태그로도 의미를 남깁니다)
- 스니펫은 위 카드의 보조 설명이므로 별도 라벨을 두지 않습니다.

## 7. 사용 예시 (의사코드)
```
{가사조각 && <LyricsSnippet segments={가사조각} />}
```

## 8. AI 지시용 스니펫
> "가사 스니펫: `#161A21` 배경, 8px 모서리, 위 카드에 4px 간격으로 붙임. 15px Pretendard 400 `#A8B0BD` 왼쪽 정렬 한 줄. 검색어 일치 부분은 `<mark>`에 `#2E7DFF` 배경 + 화이트 글자 4px 모서리(브라우저 기본 노란 배경 덮어쓰기). 줄바꿈 허용, 가로 스크롤 금지. 클릭 대상 아님."
